Build from Source

Compile MGenie from the source code on your Mac.

Requirements

  • Xcode 15.3+ — Download from the App Store
  • macOS 13+ — Build host requirement
  • Swift 5.9+ — Included with Xcode
💡 For Apple Intelligence features, you need Xcode 26+ and macOS 26 SDK.

Clone the Repository

Open Terminal and run:

git clone https://github.com/ampclaw/MGenie.git
cd MGenie

Build in Xcode

Option 1: GUI (Recommended for beginners)

  1. Open the project in Xcode: 
    open MGenie.xcodeproj
  2. Select the MGenie scheme from the top menu
  3. Select your Mac as the destination (or Any Mac for universal builds)
  4. Press ⌘B to build, or ⌘R to build and run

Option 2: Command Line

xcodebuild -scheme MGenie -configuration Release

The compiled app will be in build/Release/.

Troubleshooting

"Cannot find module" error

Clean and rebuild:

xcodebuild clean -scheme MGenie
xcodebuild build -scheme MGenie

Swift version mismatch

Verify you have Swift 5.9+:

swift --version

If out of date, update Xcode from the App Store.

Apple Intelligence features not available

⚠️ The FoundationModels framework requires Xcode 26+ and macOS 26 SDK. If you're on an older Xcode version, the AI features will be unavailable but the app will still compile.

Dependency Management

MGenie uses Swift Package Manager for dependencies. All packages are automatically resolved by Xcode.

Local dependencies:

  • HotKey/ — Global keyboard shortcuts

No additional setup is needed. Just build and run!

Create a Release Build

To create a distributable app and DMG:

xcodebuild -scheme MGenie -configuration Release archive -archivePath MGenie.xcarchive
xcodebuild -exportArchive -archivePath MGenie.xcarchive -exportOptionsPlist ExportOptions.plist -exportPath .

See the scripts/ folder in the repo for automated build scripts.
